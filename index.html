Fat Menus

An implementation of the fat-menu UX pattern.

Menu Item Number 2 is the only link in the menu that brings your somewhere (to a pivot page showing a selected menu item).

Nothing is perfect. But this is an experimentation giving me the opportunity to think outside the box. I just see this as a code kata. Let's call this one: a11y kata: Fat Menus (I am pretty good at naming stuff!)

Motivation Behind Certain Decisions

The sub-items do not appear while navigating through keyboard

I felt overwhelmed when activating sub-menus in keyboard nativation. One could argue: But au contaire! I wish to provide my fat menu to every user because it is so cool! And in any case, I provide a goto main content link in front of my fat menu. In that case: please do.

I do prefer to have a pivot page where the selected sub-menu is visible.

Keeping the fat menu inside the nav element on pivot pages

I first thought of having a copy of the selected sub-menu inside the main area. Without any wai-aria attributes do make sure the selected sub-menu is not repeated, and for simplicity of this example, I kept it in the nav element.

Bonus: as a keyboard user, I still have the possibility to skip all the navigation links and go faster to a link inside the main content of the pivot page if I so desire.
